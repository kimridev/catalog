Kimri manual catalog data checklist

Purpose
- This file is for manual data you should curate yourself.
- IDs below are intended to be stable, hardcoded app keys.
- Frontend assets/logic should branch on these IDs, not on display names.
- Do not casually rename IDs once UI/runtime logic depends on them.

Hardcoded ID policy
- Game category IDs are stable product keys.
- Variant IDs are stable product keys.
- Modding profile IDs are stable product keys.
- Display names can change later. IDs should not.
